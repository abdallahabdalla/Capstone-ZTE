Project Overview

A right-sized Zero Trust Authentication system for small and medium enterprises (SMEs). Uses Authentik as an OIDC identity provider with mandatory MFA, a Node.js/Express backend with JWT validation, and a Vite frontend dashboard — all running via Docker Compose.

Architecture

  • Client Zone — Vite vanilla JS frontend at localhost:5173. Login, token storage, refresh, logout.
  • Identity Zone — Authentik at localhost:9000. OIDC authorization, Authy TOTP MFA, RS256 JWTs with 5-min expiration.
  • Application Zone — Node.js/Express at localhost:4000. Validates every JWT via express-jwt + jwks-rsa.

Features

  • Login with MFA via Authy Authenticator (TOTP) through Authentik OIDC
  • Protected endpoint GET /secure-data requiring a valid Bearer token
  • JWT access tokens with 5-minute expiration
  • Automatic token refresh at 1 min remaining + manual refresh button
  • Logout clears tokens locally and redirects home
  • 1-hour inactivity auto-logout with 5-min warning banner
  • Docker rapid restart policy with health checks on all containers
  • 24 unit tests passing (backend + frontend)

Prerequisites

  • Docker Desktop
  • Node.js
  • Git
  • Authy Authenticator (MFA)
  • Postman (optional, API testing)

Installation & Setup

1 — Clone the repo

git clone https://github.com/abdallahabdalla/Capstone-ZTE.git
cd Capstone-ZTE

2 — Install dependencies

cd backend && npm install && cd ..
cd client  && npm install && cd ..

3 — Configure environment

# backend/.env
JWKS_URI=http://localhost:9000/application/o/my-app/jwks/
AUTHENTIK_ISSUER=http://localhost:9000/application/o/my-app/

Running the Project

# Terminal 1 — Authentik
cd ~/Documents/Capstone-ZTE/authentik-config && docker compose up -d

# Terminal 2 — Backend
cd ~/Documents/Capstone-ZTE/backend && node server.js

# Terminal 3 — Frontend
cd ~/Documents/Capstone-ZTE/client && npx vite

Then open http://localhost:5173 in your browser.

Running Tests

cd backend && npm test   # 8 backend tests
cd client  && npm test   # 16 frontend tests
